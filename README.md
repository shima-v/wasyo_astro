# 🌿 もみほぐしサロン ホームページ制作プロジェクト

「もみほぐしサロン」の公式ホームページ制作プロジェクトです。
Astro（Minimal Starter）をベースに、**スマホ特化・爆速・低コスト**をコンセプトに構築します。

## 🚀 プロジェクトの構成（Project Structure）

Astroプロジェクトの主要なフォルダとファイルは以下の通りです：

```text
/
├── public/              # 画像やfaviconなど、最適化不要な静的アセット
├── src/
│   ├── components/      # (推奨) 再利用可能なパーツ（ボタン、ヘッダー等）
│   └── pages/           # ルーティング。index.astro がトップページ
└── package.json         # プロジェクトの依存関係管理
```

*   **src/pages/**: Astroは、このディレクトリ内の `.astro` または `.md` ファイルをルートとして認識します。
*   **src/components/**: Astro/React/Vueなどのコンポーネントを配置する推奨場所です。

## 🛠 技術スタック & 実行環境

プロジェクトの安定性と開発効率を重視し、以下の構成を採用しています。

### **コア・フレームワーク**
*   **[Astro](https://astro.build/) (Static Site Generator)**
    *   静的なHTMLを優先的に出力することで、スマホでの爆速な読み込みを実現。
    *   必要最小限のJavaScriptのみをクライアントに送る「Astro Islands」アーキテクチャを活用。

### **開発・実行基盤**
*   **Node.js:** `v20.x` (Long Term Support)
    *   プロジェクトで推奨される安定版を使用。
*   **Version Manager: [fnm](https://github.com/Schniz/fnm)** (Fast Node Manager)
    *   Rust製の高速なバージョン管理ツール。`.node-version` に基づき、環境を自動で切り替え。
*   **Package Manager: [pnpm](https://pnpm.io/ja/)**
    *   `node_modules` の重複を排除し、ディスク容量の節約とインストールの高速化を実現。

### **インフラ & 外部連携（予定）**
*   **Hosting:** GitHub Pages
    *   GitHub Actions を利用し、リポジトリへのプッシュで自動デプロイ。
*   **CMS:** [microCMS](https://microcms.io/)
    *   「お知らせ」や「休日」を、コードを触らずにスマホから更新できる仕組みを導入予定。

## 🧞 コマンド一覧（Commands）

すべてのコマンドは、プロジェクトのルートディレクトリで実行してください。

| コマンド | 内容 |
| :--- | :--- |
| `pnpm install` | 依存関係（ライブラリ）のインストール |
| `pnpm dev` | ローカル開発サーバーを `localhost:4321` で起動 |
| `pnpm build` | 本番用のサイトを `./dist/` に書き出し |
| `pnpm preview` | デプロイ前にビルドした内容をローカルで確認 |
| `pnpm astro ...` | `astro add` 等のCLIコマンド実行 |

## 🐞 デバッグ方法

Astroはコードが実行される「場所」によってデバッグ方法が異なります。

### 1. サーバーサイド・デバッグ
`src/pages/*.astro` のフロントマター（`---` に囲まれた部分）に記述したコード。
*   **方法:** `console.log(変数)` を記述。
*   **出力先:** VS Codeの **ターミナル** に表示されます。ブラウザ側には出ません。
*   **用途:** APIデータの取得確認や、ビルド時の条件分岐のチェック。

### 2. クライアントサイド・デバッグ
`<script>` タグ内に記述した、ブラウザ上で動くJavaScript。
*   **方法:** `console.log()` または `debugger;` を記述。
*   **出力先:** ブラウザの **開発者ツール (F12) > Console** に表示されます。
*   **用途:** ボタンのクリック、メニューの開閉など、ユーザー操作のチェック。

## 💡 実装のポイント（スマホ特化 & SEO）

*   **ローカルSEO:** タイトルタグには必ず **「地域名 + 屋号 + もみほぐし」** を含める。
*   **アクセシビリティ:** スマホでの誤操作を防ぐため、タップターゲット（ボタン等）は **44px × 44px** 以上を確保。
*   **画像最適化:** `astro:assets` を使用して画像をWebP形式に自動変換し、読み込み速度を最大化する。
*   **コンバージョン導線:** 画面下部に「LINE予約」または「電話」のボタンを `position: fixed;` で常駐させる。
